## 2. Day3 产品策略（为什么这次要强调“审美资产化”）

1. InviteVerse 的差异化不只是“能生成邀请函”，而是“能持续生成有审美主张、可复用、可扩展的邀请体验”。
2. 审美资产化目标：把主理人的设计经验写成机器可调用规则，而不是每次从零拼页面。
3. 规则拆层：
   1. Theme DNA：场景、关系强度、情绪密度、私密感。
   2. Template Rules：布局、字体、色板、玻璃质感、动效曲线。
   3. Runtime Spec：最终渲染字段（event/copy/tokens/actions）。
4. 业务价值：
   1. 同一主题可批量复用。
   2. 不同主题可稳定扩展。
   3. LLM 输出受规则约束，降低“风格漂移”。

## 3. Demo1 升级目标（本次重构范围）

1. 主题从“情侣三周年纪念”改为“情侣一周年纪念”。
2. 叙事从一句话升级为完整四幕故事：

   2. 解锁具体赴约信息（时间地点与二人备注）。
   3. 校园故事正文（相识、相恋、宿命高光）。
   4. 结尾确认赴约（情绪闭环）。
3. 内容素材紧扣设定：情侣相识于中国科大大一阶段，保留校园记忆与命运感。
4. 视觉与动效目标：
   1. 保留温暖浪漫。
   2. 提升层次与质感。
   3. 减少“空页感”，增强一体化叙事体验。

## 4. Demo1 详细实现拆解（重点）

### 4.1 主题语义定义

1. 关键词：温馨、私密、命定、校园回忆、郑重邀请。
2. 情绪节奏：
   1. 首屏克制（安静进入）。
   2. 中段揭示（信息解锁）。
   3. 故事升温（高光时刻串联）。
   4. 结尾收束（确认赴约）。

### 4.2 视觉系统（Color / Type / Material）

1. 色彩：以奶油粉、暖杏、香槟金为主，避免高对比硬边，保持晚霞与烛光气质。
2. 字体：`Cormorant Garamond + Noto Serif SC`，兼顾优雅拉丁字形与中文正文可读性。
3. 材质：玻璃拟态卡片（毛玻璃 + 轻边框 + 内高光）用于承载“私密信封”语义。
4. 背景层次：
   1. 缓慢漂移的暖色渐变底。
   2. 全屏模糊层。
   3. 细颗粒纹理层。
   4. 指针暖光层（跟随鼠标/触控移动）。

### 4.3 动效系统（Motion Choreography）

1. 首屏卡片入场：
   1. `y: 20 -> 0`。
   2. `opacity: 0 -> 1`。
   3. 时长 `1.5s`。
   4. 曲线 `cubic-bezier(0.25, 1, 0.5, 1)`。
2. 卡片呼吸：首屏入场后叠加轻微呼吸，维持生命感但不过度戏剧化。
3. 解锁反馈：滚动触发 `is-unlocked`，详情卡遮罩上移形成“揭封”体验。
4. 页面推进：每个内容区块 `IntersectionObserver` 分段显现，避免一次性堆砌信息。
5. 氛围微动：漂浮花瓣 + 指针光晕 + 向上提示箭头，形成持续但克制的动态环境。

### 4.4 内容系统（故事与信息设计）

1. 首屏文案：明确“一周年”与回到科大场景，建立时间和地点锚点。
2. 赴约信息卡：时间、地点、二人备注三项完整，强化“只属于两个人”的私密感。
3. 校园故事正文：
   1. 起点：大一新生周在东区三教相识。
   2. 发展：图书馆闭馆前互改作业、操场流星愿望、实验室凌晨同行。
   3. 定义：通过“我们不是偶然”强化宿命感。
4. 高光时刻模块：三条时间轴卡片（2024·09 相识 / 2025·03·15 确定关系 / 2026·03 一周年）让故事具象可感。
5. 结尾 CTA：情绪金句 + 日期确认 + “确认赴约”按钮，完成体验闭环。

### 4.5 交互与可用性

1. 移动端优先适配：卡片圆角、字号、行高、分栏在小屏下降级为单列。
2. 降噪策略：采用单一暖色系和少量交互点，避免“炫技型噪音”。
3. 可访问性：
   1. 保留语义化 section/aria-label。
   2. 提供 `prefers-reduced-motion` 降级。
   3. 交互反馈通过 toast 告知状态。

## 7. Demo2 目标定义（赛博朋克生日派对）

设计目标：
1. 先产出“中文版”，提升阅读效率：除主标题 `CYBER BIRTHDAY PARTY` 保留英文外，其他标题与正文统一中文表达。
2. 内容从“短宣言”升级到“任务简报”：身份设定、主题动机、任务详情、交互按钮一体化。
3. 视觉严格贴合 Cyberpunk 2077 调色板：赛博黄 `#FCEE0A`、霓虹青 `#00F0FF`、故障红 `#FF003C`、深渊黑 `#0B0B0C`。
4. 交互上落地进场序列、glitch 主标题、毫秒级时钟、机械按压反馈与背景光雨。

技术落地策略：
1. 先按“单文件 React”方案实现，便于复制演示。
2. 线上验证时发现部分环境会出现 ESM 外链加载失败，导致正文不渲染（仅剩背景层）。
3. 为保证可用性，最终改为“本地静态可运行版”（HTML + CSS + JS），彻底去除运行时外链依赖。
4. 通过 CSS 伪元素 + `clip-path` + 微随机位移实现主标题 glitch；通过 JS 实现序列显现与按压反馈。

## 8. Demo2 详细实现拆解（已完成）

### 8.1 背景系统（黑底、网格、CRT）

1. 黑底：页面基础背景固定为纯黑。
2. 网格：双层 `linear-gradient` 生成赛博网格，透明度控制在 0.32，避免喧宾夺主。
3. CRT 扫描线：`repeating-linear-gradient` 构造横向扫描线，并加入缓慢漂移。
4. 工业噪声：叠加轻量 noise 纹理层，营造旧屏幕信号失真感。
5. 暗角：vignette 层压暗边缘，聚焦中区信息。

### 8.2 排版系统（Brutalism + 中文可读）

1. 主标题：`CYBER BIRTHDAY PARTY`（英文保留，作为视觉锚点）。
2. 其余文案：统一中文任务简报风格，降低理解门槛。
3. 字体策略：全局 `font-mono`，字重偏粗，工业感优先。
4. 版式策略：状态栏 -> 英文大标题 -> 中文协议覆盖行 -> Mission Intel 卡 -> CTA。

### 8.3 核心 Glitch 视效（严格执行）

1. 主标题使用 `::before`、`::after` 两个伪元素复制文本。
2. 两层分别使用青色/品红色，形成 RGB 通道撕裂感。
3. 通过 `clip-path: inset(...)` 关键帧切片，制造撕裂扫描错位。
4. JS 每 120ms 随机更新 `--glitch-x / --glitch-y`，叠加高频随机位移。
5. 随机短暂亮度脉冲（flicker）强化“故障瞬断”。

### 8.4 按钮交互（机械感）

1. CTA 使用矩形硬边按钮，不使用圆角胶囊语义。
2. 主按钮初始态：赛博黄底黑字；hover 变霓虹青并伴随瞬时故障抖动。
3. 次按钮语义明确：拒绝协议（故障红边框）。
4. 点击反馈：`pointerdown` 添加 `is-pressed`，`scale(0.95)` 下压感明显。
5. 点击后 toast 反馈：分别对应“接入网络/拒绝协议”。

### 8.5 内容结构（赛博派对语境）

1. 顶部状态栏：`[SYS_UPDATE] CYBER BIRTHDAY PARTY`。
2. 协议覆盖行：`>_ 协议覆盖：[Lucas] 迎来了他的 20 岁生日。`。
3. 主理人设定（最终文案）：`我是Lucas, 5 月 31 日是我 20 岁生日。想邀请我的朋友们一起登上外滩游艇, 把这一晚过成真正的赛博生日局!`。
4. Mission Intel 卡片字段（完整落地）：
   1. `目标 TARGET`：20 岁生日派对。
   2. `时间 TIME_SYNC`：`2026.05.31 // 19:00:00` + 毫秒级跳动时钟。
   3. `坐标 COORD`：`上海外滩码头 登陆游艇`。
   4. `装备要求 DRESS_CODE`：`赛博穿搭，怎么炫酷怎么来`。
5. 精简策略（最终确认）：
   1. 已按产品决策删除 `YACHT ZONES`、`NIGHT SCRIPT`、`CREW PROTOCOL` 三个模块。
   2. 页面信息密度聚焦在“核心邀约信息 + 交互按钮”，降低阅读负担。
6. CTA 区：
   1. 主按钮：`[ 接入网络 // JACK IN ]`。
   2. 次按钮：`[ 拒绝协议 // DECLINE ]`。
